# איקס עיגול (Tic-Tac-Toe)

משחק איקס-עיגול בדפדפן. נבנה ב-**React + TypeScript + Vite**, צד-לקוח בלבד (ללא שרת).

## תכונות
- **דף בית** שבו בוחרים לפני המשחק: מצב משחק, רמת קושי, ושמות השחקנים, ואז "התחל משחק".
- **שני מצבי משחק:**
  - **נגד שחקן (PvP)** — שני שחקנים על אותו מסך.
  - **נגד מחשב (PvC)** — מול AI שחוזקו משתנה לפי רמת הקושי: קל = מהלכים אקראיים,
    בינוני = ניצחון/חסימה, קשה = **minimax** (בלתי מנוצח).
- **טיימר לכל מהלך** עם 3 רמות קושי (קל 15ש' / בינוני 10ש' / קשה 5ש'). אם הזמן נגמר —
  השחקן מאבד את התור והמהלך עובר ליריב. פס התקדמות ויזואלי (אדום ב-3 השניות האחרונות).
  במצב נגד מחשב הטיימר רץ רק בתור השחקן האנושי.
- **שמות שחקנים** המופיעים בתצוגת התור ובהודעת הניצחון (נגד מחשב — היריב הוא "מחשב").
- זיהוי ניצחון בכל 8 הקווים, עם **הדגשת הקו המנצח**, וזיהוי תיקו.
- חסימת לחיצה על משבצת תפוסה ואחרי סיום המשחק.
- כפתורי "משחק חדש" (אתחול עם אותן הגדרות) ו-"חזרה לדף הבית".
- נגישות: כפתורים אמיתיים, `aria-label` לכל משבצת, `role="timer"`, ניווט מקלדת, ופריסת RTL.

## הרצה

```bash
npm install        # התקנת תלויות
npm run dev        # הרצת שרת הפיתוח (http://localhost:5173)
npm run build      # בנייה לפרודקשן (כולל בדיקת טיפוסים)
```

## בדיקות

```bash
npm test           # Jest — unit (לוגיקה) + component (אינטגרציה)
npm run e2e        # Playwright — בדיקות end-to-end
npm run e2e:ui     # Playwright במצב UI אינטראקטיבי
```

## מבנה הקוד

```
src/
  game/                    # לוגיקה טהורה — מנותקת מ-React, נבדקת ב-Jest
    gameLogic.ts           #   זיהוי ניצחון/תיקו
    difficulty.ts          #   קונפיג רמות הקושי (משכי הטיימר)
    ai.ts                  #   יריב המחשב (אקראי / היוריסטי / minimax)
    gameMode.ts            #   טיפוסי מצב המשחק + GameConfig
  hooks/useCountdown.ts    # hook ייעודי לספירה לאחור (מנותק מ-UI)
  components/
    HomeScreen.tsx         # דף הבית — בחירת מצב/קושי/שמות
    GameScreen.tsx         # מסך המשחק — state, טיימר, ושילוב ה-AI
    Board.tsx / Square.tsx # הלוח והמשבצות
    GameStatus.tsx         # תצוגת תור / מנצח / תיקו
    TurnTimer.tsx          # פס התקדמות הטיימר
    ModeSelector.tsx       # בחירת מצב משחק
    DifficultySelector.tsx # בחירת רמת קושי
    PlayerSetup.tsx        # שדות הזנת שמות (1 או 2 לפי המצב)
  App.tsx                  # נתב דק בין דף הבית למשחק
e2e/game.spec.ts           # בדיקות Playwright
docs/                      # PLAN*.md (תכנון) + TEST_CASES.md (מקרי בדיקה)
```

### עקרון תכנון מרכזי
כל הלוגיקה ([gameLogic.ts](src/game/gameLogic.ts), [ai.ts](src/game/ai.ts),
[difficulty.ts](src/game/difficulty.ts)) כתובה כפונקציות **טהורות** ללא תלות ב-React או
ב-DOM — מה שמאפשר בדיקות יחידה פשוטות וממוקדות ומפריד בבירור בין לוגיקה לתצוגה.
[App.tsx](src/App.tsx) הוא נתב דק; ה-state של המשחק חי ב-[GameScreen.tsx](src/components/GameScreen.tsx)
בלבד, וערכים נגזרים (מנצח, תיקו) מחושבים מחדש בכל רינדור ולא נשמרים.

ראו [docs/TEST_CASES.md](docs/TEST_CASES.md) לרשימת מקרי הבדיקה המלאה.
